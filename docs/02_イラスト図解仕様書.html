コンセプト

すべては、ひとつの木

「定跡を作る」「仕掛けの局面を記録する」「問題集で覚える」—— 別々に見える機能は、じつは同じ形をしています。ある盤面（＝局面ノード）を、手（＝枝）でつないでいくと、 初期局面を根とした一本の木になります。

戦法や変化はその枝分かれ。棋譜は木の中の一本の道。問題は、木のどこかを指して「次の一手は？」と問う印。 だから、作ったものがそのまま記録になり、記録がそのまま問題になります。

3つの目的

作る・記録する・覚える

作る

初期局面から手を指して、定跡の木を育てる。分岐もコメントも自由に足せる。

記録する

仕掛けが成立する局面を、木の一点として保存。任意のノードがそのまま記録になる。

覚える

木のノードから「1手当て」の問題を作成。間違えた手は、また出題される。

どこでも使える

スマホと PC で、同じ一冊

ブラウザで動くアプリ（PWA）なので、スマホでも PC でも同じ画面が開けます。ログインすれば、どの端末でも常に最新。 通勤中にスマホで作った定跡の続きを、家では PC の大きな画面で——という使い方ができます。

画面イメージ

4つの画面

盤面で手を指す動かせるマスがハイライトされ、木に手が足されていく。
樹形図で見渡す木の全体を俯瞰。ノードをタップすれば、その局面へ。
1手を当てる局面に対して正しい一手を選ぶ。分岐は「どれでも正解」も選べる。
ノートブックで整理種別ごとに問題をまとめる。作成・編集・削除も自由。

学びの流れ

解いて、身につける

出題から復習までは一本道。間違えた問題はプールに戻り、正解できるまで、また巡ってきます。

技術構成

仕様まとめ

React + TypeScript Vite / PWA shogiops（将棋ルール） 自作の盤 UI Supabase（DB・認証・同期） スマホ・PC 両対応

ブラウザで動く PWA を React／TypeScript で作り、将棋のルール判定は実績あるライブラリ（shogiops）に任せる。 盤の描画とハイライトは自作。データ・ログイン・同期は Supabase（Postgres）が受け持つ。 個人利用なら、維持コストはかからない構成。
