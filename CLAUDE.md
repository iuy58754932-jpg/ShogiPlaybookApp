# ShogiPlaybookApp — プロジェクトメモリ

将棋の定跡・問題集アプリ（仮称「定跡ツリー」）。定跡を作り、仕掛けが成立する局面を記録し、
1手当ての問題集で覚える。中核は「すべては1つの木」——局面ノードを手でつないだ木構造。
棋譜・問題・ノートブックはすべてこの木を参照する。

## 技術スタック

- React + TypeScript + Vite（PWA化: vite-plugin-pwa）
- 将棋ルール: shogiops（合法手・打ち駒の生成、妥当性チェック、SFEN/USI 入出力。ライセンス GPL-3.0）
- 盤UIは自作（9×9グリッド＋駒コンポーネント）。ルール判定は自前実装しない
- バックエンド: Supabase（Postgres + Auth + Row Level Security）
- ホスティング: Vercel / Netlify / Cloudflare Pages の無料枠

## 進め方（重要）

- docs/01 の「9. 推奨ビルド順」に従い、**Phase 0 から1フェーズずつ**実装する。
- 各フェーズ完了時に、その時点で動く状態を保つ（インクリメンタルに積み上げる）。
- 実装に着手する前に、まず作業計画を提示して確認を取ること。

## 方針

- SFEN を局面の一意キーとして使う。
- 合法手・反則（二歩・行き所のない駒・打ち歩詰め・王手放置）の判定は shogiops に任せる。
- 秘密情報（Supabase の service_role キー等）はフロントに置かず、コミットしない。`.env` を使い `.gitignore` する。フロントで使うのは anon キーのみ。
